# PROMPT — для Claude в VS Code

Привет. Соберём вместе мобильное web-приложение ресторана «Сказка Востока» на React + Vite + TypeScript.

## Что у тебя есть

- `BRIEF.md` — полное описание проекта: цели, дистрибуция, фичи, модель данных, роуты, архитектура папок, критерии приёмки. **Прочитай его первым, полностью, до того как начнёшь что-либо делать.**
- `design-system/` — готовая дизайн-система: палитра (CSS-переменные на `:root`), типографика (Alice + Roboto Flex), базовые компоненты. Используй её как источник истины для всех визуальных решений. Не придумывай свои цвета, размеры, радиусы — только из дизайн-системы.
- `design-references/` — скриншоты существующего прототипа и фото блюд ресторана. Это референсы настроения и структуры экранов, а не макеты для копирования пиксель-в-пиксель.

## Правила работы

1. **Дизайн-система — закон.** Любой цвет — через CSS-переменную. Любой размер текста — из типографической шкалы. Любой отступ — из spacing-шкалы. Если чего-то не хватает — предложи расширение шкалы, не изобретай локально.

2. **Моки сейчас, API потом.** Пока в `BRIEF.md` в разделе API Contract стоит заполнитель `<ВСТАВИТЬ СПЕКУ>` — работай на моках из `src/mocks/`. Вся работа с данными должна идти через слой `src/api/*` — когда API появится, поменяется только реализация внутри этих файлов, компоненты не затрагиваются.

3. **TypeScript везде.** Никакого `any`. Типы из `BRIEF.md` (Dish, CartItem, Order и т.д.) вынеси в `src/types/`.

4. **Компоненты маленькие.** Один компонент — одна ответственность. Если файл больше 200 строк — декомпозируй.

5. **Не пиши длинных объяснений в ответах.** Короткий план → код → короткое резюме «что сделал, что дальше». Я читаю код, а не прозу.

6. **Спрашивай, если не уверен.** Лучше один уточняющий вопрос, чем час переделок. Особенно по бизнес-логике (модификаторы, расчёт цены весовых, применение акций).

**Остановись. Покажи структуру и базовый `App.tsx` с заглушкой.**

- Zustand-стор `user` с состоянием авторизации

**Остановись. Проверю в трёх окружениях.**

## Что делать нельзя

- Не используй `any` в TypeScript
- Не хардкодь цвета, шрифты, размеры — только через токены дизайн-системы
- Не создавай компоненты «на вырост» — делай только то, что нужно сейчас
- Не подключай лишние библиотеки (styled-components, mui, antd и т.д.) — стек зафиксирован
- Не пиши бэкенд-логику на клиенте — всё идёт через `src/api/*`
- Не делай localStorage-костыли там, где должно быть состояние (Zustand) или серверные данные (TanStack Query)
- Не игнорируй фазы — не собирай всё сразу в один коммит

- Предлагай улучшения, если видишь, что в брифе что-то нелогично
