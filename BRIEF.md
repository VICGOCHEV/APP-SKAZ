# BRIEF — Сказка Востока

Мобильное web-приложение ресторана восточной/кавказской кухни «Сказка Востока» для заказа еды с доставкой и самовывозом.

## Контекст и дистрибуция

Одна кодовая база, четыре канала дистрибуции через web-view:

- **Telegram Mini App** — основной канал, авторизация через Telegram SDK
- **VK Mini App** — авторизация через VK Bridge
- **MAX** — web-view
- **Браузер + обёртка для iOS/Android стора** — SMS-авторизация

Приложение определяет окружение автоматически и адаптирует поведение авторизации, шаринга и системных кнопок.

## Метафора продукта

Скатерть-самобранка на восточный лад. Гостеприимство, щедрость, обилие блюд, разложенных перед гостем. Ключевой визуальный герой — фотографии блюд, снятые сверху в едином фирменном стиле. Дизайн подчёркивает фото, а не соревнуется с ними.

## Целевая платформа

- Mobile-first, viewport от 360px
- Основной целевой размер — 390px (iPhone 14/15)
- Адаптив до планшета/десктопа — вторично, но не ломается

## Стек

- **React 18** + **Vite** + **TypeScript**
- **React Router v6** — клиентский роутинг
- **Tailwind CSS** — стилизация
- **lucide-react** — иконки
- **Zustand** — состояние (корзина, пользователь, UI)
- **TanStack Query** — кеш запросов к API
- **react-hook-form** + **zod** — формы и валидация
- **framer-motion** — анимации (сторис, bottom sheet, переходы)

## Дизайн-система

Отдельный артефакт из claude.ai/design, который прикладывается к проекту. Из него берутся:

- CSS-переменные палитры (`:root`)
- Типографическая шкала (Alice для заголовков и цены, Roboto Flex для остального — оба через Google Fonts)
- Базовые компоненты (кнопки, инпуты, карточки, ползунок веса, таб-бар, сторис, баннер, bottom sheet, группированные списки)

## Авторизация

Абстракция `AuthProvider`, которая определяет окружение:

- `window.Telegram?.WebApp` → Telegram auto-login (initData + hash-проверка на бэкенде)
- `window.vkBridge` → VK Bridge auto-login
- Fallback → экран входа по телефону + SMS-код

Гостевой режим: до момента оформления заказа можно смотреть меню и собирать корзину без авторизации. Корзина сохраняется в localStorage и синкается с сервером после входа.

## Бэкенд

Готовый API со стороны сайта ресторана, который уже интегрирован с системой управления IKKO. Спецификация API вставляется в раздел «API Contract» ниже.

До получения спеки — работаем на моках в `src/mocks/` (блюда, категории, модификаторы, стоп-лист, акции, заказы).

## Функциональные требования

### Точки входа

Первый экран — всегда **Главная**. Онбординг и обязательный логин отсутствуют. Логин запрашивается в момент оформления заказа или открытия Профиля.

### Главная

Сверху вниз:

1. **Сторис** — горизонтальная лента круглых превью ~64px. Тап → полноэкранный просмотрщик с автопрогрессом 5 сек, тап по правой половине → вперёд, по левой → назад, свайп вниз → закрыть. Внутри сторис — опциональная CTA-кнопка (deeplink на блюдо / категорию / акцию / внешнюю ссылку). Состояние «просмотрено» хранится на сервере (синхронизируется между устройствами).
2. **Баннер-слайдер** — горизонтально свайпаемая карусель с автопрокруткой 5–7 сек, точки-индикаторы. Каждый слайд — deeplink.
3. **Поиск блюда** — строка с живым поиском по названию и составу.
4. **Цветные плитки-ярлыки** — `история заказов`, `меню кухни`, `Песах` (сезонное меню), `заказ стола`.
5. **Категории меню** — горизонтальный скроллер с круглыми фото ~72px и подписью.
6. **Часто заказывают** — сетка 2 колонки с карточками блюд.

### Меню категории

Заголовок категории, сверху — горизонтальные табы-чипсы подкатегорий. Сетка 2 колонки. Карточка: фото (или заглушка с иконкой ножа и вилки, если в IKKO нет картинки), название, граммовка, краткий состав, цена, кнопка «+». Тап по карточке → Карточка блюда.

### Карточка блюда

Открывается как полноэкранный bottom sheet поверх текущего экрана.

Блоки: большое фото, название (Alice), цена (Alice), граммовка текущей порции, время приготовления (`20–30 мин`), время доставки (`30–45 мин`), раскрываемый блок **Состав**, контрол веса/количества, опциональные модификаторы (без лука, острое/неострое), sticky-кнопка снизу `Добавить в корзину` с итоговой ценой.

**Два режима контрола:**

- **Весовые блюда** — ползунок граммовки с пресетами-пилюлями (100/200/300/500г), шаг 50г. Цена пересчитывается динамически. Один добавляемый инстанс = выбранный вес.
- **Штучные блюда** — обычный степпер количества.

Флаг `isWeighted` на блюде определяет режим.

### Корзина

Таб в таб-баре. Позиции редактируемы прямо в строке (вес/количество). Сумма пересчитывается. Промокод/бонусы. Кнопка «Оформить заказ».

### Оформление заказа

- Выбор способа: доставка / самовывоз (самовывоз автоматически применяет акцию «–20%»)
- Адрес доставки (автодополнение, сохранённые адреса из профиля)
- Время (ASAP / ко времени)
- Способ оплаты (карта онлайн / наличные / картой курьеру)
- Комментарий
- Применение промокода / бонусов
- Отправка в IKKO через API, получение номера заказа, статус

### Статус заказа

Экран с прогрессом: принят → готовится → передан курьеру → в пути → доставлен. Опционально push-уведомления (в мини-аппах через SDK, на вебе — через Web Push).

### Акции

Отдельный таб. Список карточек: фото, заголовок, описание. Часть акций применяется автоматически при оформлении, часть — через промокод.

### Профиль

Группированный список:

- **Контакт** — телефон + кнопка «Редактировать»
- **Платёжные карты** — привязка/удаление, оплата в один тап
- **История** — история заказов (тап → детали, «повторить заказ» одним тапом кладёт всё в корзину), адреса доставки
- **Дополнительно** — уведомления, о приложении, **Книга отзывов** (открывает чат с ИИ-ассистентом)
- **Бонусы** — счётчик накопленных баллов
- **Выйти**

### Чат с ИИ-ассистентом

Отдельный экран с историей переписки, полем ввода, quick-replies (жалоба / похвала / вопрос по блюду / вопрос по заказу). Интеграция — отдельный endpoint на бэкенде.

### Таб-бар

`Меню` · `Главная` · `Акции` · `Корзина` (с бейджем количества) · `Профиль`. Доступен со всех основных экранов. Скрывается на полноэкранных модалках (карточка блюда, чат, чекаут, просмотр сторис).

## Нефункциональные требования

- **Производительность** — LCP <2.5с, картинки блюд через CDN в WebP с размерами под retina/не-retina, lazy-loading ниже фолда
- **Доступность** — контраст AA, фокус-стили, семантический HTML, ARIA для кастомных контролов (ползунок, вкладки, bottom sheet)
- **Тёмная тема** — не в MVP, но все цвета через CSS-переменные, чтобы добавить позже одним файлом
- **Аналитика** — события `story_view`, `story_cta_click`, `banner_click`, `dish_view`, `add_to_cart`, `checkout_start`, `order_placed` с параметрами (id, цена, категория). Провайдер — настраиваемый (Яндекс.Метрика, Telegram Analytics)
- **i18n** — архитектурно заложено (все строки через `t('key')`), но в MVP только русский

## Что НЕ входит в MVP

- Тёмная тема (только архитектурно)
- Мультиязычность (только инфраструктура)
- Реферальная программа
- Оплата криптой, СБП — только в V2
- Нативные приложения (iOS/Android) — обёртка вокруг веба для сторов только в V2

## Структура данных (мок-уровень)

```ts
type Dish = {
  id: string
  name: string
  description: string
  composition: string      // состав
  photoUrl: string | null  // null → показать заглушку
  categoryId: string
  cuisineId: string
  isWeighted: boolean
  price: number            // для штучных — за штуку, для весовых — за baseWeight
  baseWeight: number       // для весовых (обычно 100г)
  weightPresets: number[]  // [100, 200, 300, 500]
  cookTime: string         // "20-30 мин"
  deliveryTime: string     // "30-45 мин"
  modifiers: Modifier[]
  allergens: string[]
  isAvailable: boolean     // стоп-лист из IKKO
}

type Category = { id: string; name: string; photoUrl: string; order: number }
type Cuisine  = { id: string; name: string; order: number }

type Story  = { id: string; previewUrl: string; slides: Slide[]; seen: boolean }
type Slide  = { type: 'image' | 'video'; url: string; cta?: { label: string; deeplink: string } }
type Banner = { id: string; imageUrl: string; title: string; subtitle: string; deeplink: string }

type CartItem = {
  dishId: string
  quantity: number         // для штучных
  weight?: number          // для весовых
  modifiers: string[]      // id выбранных модификаторов
  price: number            // посчитанная цена этого инстанса
}

type Order = {
  id: string
  items: CartItem[]
  total: number
  delivery: 'delivery' | 'pickup'
  address?: Address
  time: 'asap' | string    // ISO
  payment: 'card_online' | 'cash' | 'card_courier'
  status: 'accepted' | 'cooking' | 'ready' | 'on_way' | 'delivered' | 'cancelled'
  createdAt: string
}
```

## Критерии приёмки MVP

- Гость может открыть приложение, полистать меню, добавить блюда в корзину (включая весовые с ползунком), оформить заказ (после авторизации), увидеть статус
- Авторизация работает в трёх средах: TG Mini App, VK Mini App, браузер (SMS)
- Все экраны адаптивны от 360px
- Все картинки lazy-loaded, placeholder для блюд без фото работает
- Стоп-лист IKKO корректно дизейблит недоступные блюда
- Дизайн-система используется консистентно — нет хардкодных цветов и шрифтов вне `:root`
- Навигация и переходы плавные, без белых вспышек
